Creating a Kind Cluster and Installing NGINX Ingress

In this tutorial, we will walk through the steps to create a Kubernetes cluster using kind and install the NGINX Ingress controller.

Prerequisites

  • Docker installed on your local machine
  • kubectl installed on your local machine
  • kind installed on your local machine

Step 1: Create a Kind Cluster

First, create a configuration file for the kind cluster:

cat < kind-config.yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
- role: control-plane
- role: worker
EOF

Now, create the cluster using the configuration file:

kind create cluster --config kind-config.yaml

Step 2: Verify the Cluster

Use the following command to verify that your cluster is up and running:

kubectl get nodes

Step 3: Install NGINX Ingress Controller

Apply the NGINX Ingress controller manifests:

kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml

Step 4: Verify NGINX Ingress Installation

Ensure that the NGINX Ingress controller pods are running:

kubectl get pods -n ingress-nginx

Conclusion

You have successfully created a kind cluster and installed the NGINX Ingress controller. You can now start deploying your applications and expose them using the NGINX Ingress.

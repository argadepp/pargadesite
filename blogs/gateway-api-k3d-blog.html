WHY

Gateway API vs Ingress

Ingress has served us well, but it was designed when Kubernetes routing was simpler. It has no concept of roles, forces complex behaviour into annotations, and only speaks HTTP/HTTPS. Gateway API is the official successor — it's expressive, role-oriented, and supports TCP, TLS, gRPC, and HTTP natively.

Feature Ingress Gateway API
Role separation (Infra / App / Route) ✗ None ✓ First-class
Protocol support HTTP / HTTPS only HTTP, TCP, TLS, gRPC
Traffic splitting / canary Annotation hacks Native weights
Multi-namespace routing Limited ReferenceGrant
Conformance testing ✗ None ✓ Official suite
Portability across controllers Low — annotation-dependent High — spec-driven
Gateway API is GA since Kubernetes 1.28. It's not experimental — it's the future of Kubernetes traffic management and the direction the ecosystem is moving.

STEP 01

Install k3d

k3d wraps k3s inside Docker containers, giving you a full Kubernetes cluster with proper port mapping to localhost. Unlike running k3s directly in WSL, networking just works.

bash install 
# Install k3d via official installer
curl -s https://raw.githubusercontent.com/k3d-io/k3d/main/install.sh | bash

# Verify
k3d version
expected output
k3d version v5.8.x
k3s version v1.31.x (default)
STEP 02

Create the cluster

We create the cluster with two key flags: port mappings so localhost:8080 reaches the cluster, and Traefik disabled so we can install v3 fresh with Gateway API support.

bash create cluster 
k3d cluster create gateway-demo \
  --port "8080:80@loadbalancer" \
  --port "8443:443@loadbalancer" \
  --k3s-arg "--disable=traefik@server:0" \
  --agents 1

What each flag does

FlagPurpose
--port "8080:80@loadbalancer" Maps localhost:8080 on your machine to port 80 on the k3d loadbalancer container
--port "8443:443@loadbalancer" Same for HTTPS — useful for TLS setups later
--disable=traefik@server:0 Skips the bundled Traefik v2 so we can install v3 with Helm
--agents 1 Adds one worker node (1 server + 1 agent = realistic local cluster)
bash verify 
kubectl get nodes
# NAME                         STATUS   ROLES                  AGE
# k3d-gateway-demo-server-0    Ready    control-plane,master   30s
# k3d-gateway-demo-agent-0     Ready    <none>                 20s
STEP 03

Install Gateway API CRDs

k3s does not ship Gateway API CRDs. We need to install them manually. Use v1.5.1 — this is critical because Traefik v3.7+ requires exactly this version. Installing an older version (like v1.2.1) will cause Traefik to fail with BackendTLSPolicy not found errors and the GatewayClass will stay Unknown forever.

Version mismatch is the #1 failure point. Always match the CRD version to what your Traefik chart requires. Traefik's Helm install notes will tell you the exact version needed.
bash install CRDs 
# Standard channel — HTTPRoute, Gateway, GatewayClass, etc.
kubectl apply -f https://github.com/kubernetes-sigs/gateway-api/releases/download/v1.5.1/standard-install.yaml

# Experimental channel — BackendTLSPolicy (required by Traefik v3.7+)
kubectl apply -f https://github.com/kubernetes-sigs/gateway-api/releases/download/v1.5.1/experimental-install.yaml
bash verify CRDs 
kubectl get crd | grep gateway.networking.k8s.io
expected output
backendtlspolicies.gateway.networking.k8s.io
gatewayclasses.gateway.networking.k8s.io
gateways.gateway.networking.k8s.io
grpcroutes.gateway.networking.k8s.io
httproutes.gateway.networking.k8s.io
referencegrants.gateway.networking.k8s.io
STEP 04

Install Traefik v3 with Gateway API

We use Helm to install Traefik v3 with the kubernetesGateway provider explicitly enabled. This is what registers the GatewayClass and starts reconciling your Gateway resources.

bash helm install 
# Add Traefik Helm repo
helm repo add traefik https://traefik.github.io/charts
helm repo update

# Install Traefik with Gateway API provider enabled
helm install traefik traefik/traefik \
  --namespace kube-system \
  --set providers.kubernetesGateway.enabled=true \
  --set gateway.enabled=true \
  --set service.type=LoadBalancer
bash verify 
# Wait for Traefik to be ready
kubectl rollout status deployment/traefik -n kube-system

# Check GatewayClass — should show Accepted: True
kubectl get gatewayclass
expected output
NAME CONTROLLER ACCEPTED AGE
traefik traefik.io/gateway-controller True 45s
Accepted: True means Traefik has registered itself as the controller for the traefik GatewayClass and is ready to reconcile Gateway resources.

Understand Traefik's port mapping

This is the key insight for making routing work locally. Traefik's internal entrypoint for HTTP is port 8000, not port 80. Your Gateway listener must match this internal port.

localhost:8080 → (k3d maps) → k3d loadbalancer :80
↓
Traefik web entrypoint :8000 ← Gateway port must match this
↓
HTTPRoute → Service → Pods
STEP 05

Deploy the demo app

We use traefik/whoami — a tiny Go server that reflects request headers and connection info. Perfect for verifying routing.

yaml app.yaml 
apiVersion: v1
kind: Namespace
metadata:
  name: demo
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: whoami
  namespace: demo
spec:
  replicas: 2
  selector:
    matchLabels:
      app: whoami
  template:
    metadata:
      labels:
        app: whoami
    spec:
      containers:
      - name: whoami
        image: traefik/whoami
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: whoami
  namespace: demo
spec:
  selector:
    app: whoami
  ports:
  - port: 80
bash apply 
kubectl apply -f app.yaml
kubectl rollout status deployment/whoami -n demo
STEP 06

Create Gateway and HTTPRoute

The Gateway defines where traffic enters. The HTTPRoute defines where it goes. These are separate objects owned by different teams in production — infrastructure owns the Gateway, the app team owns the HTTPRoute.

yaml gateway.yaml 
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: main-gateway
  namespace: demo
spec:
  gatewayClassName: traefik
  listeners:
  - name: http
    port: 8000       # Traefik's internal web entrypoint — NOT 80
    protocol: HTTP
    allowedRoutes:
      namespaces:
        from: Same
port: 8000, not 80. This is the most common mistake. The Gateway listener port must match Traefik's internal entrypoint port (--entryPoints.web.address=:8000), not the host-level port you use in curl.
yaml httproute.yaml 
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: whoami-route
  namespace: demo
spec:
  parentRefs:
  - name: main-gateway
    namespace: demo
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /
    backendRefs:
    - name: whoami
      port: 80
bash apply and verify 
kubectl apply -f gateway.yaml
kubectl apply -f httproute.yaml

# Check status of both resources
kubectl get gateway,httproute -n demo

# Gateway should show Programmed: True
kubectl describe gateway main-gateway -n demo
expected output
NAME CLASS ADDRESS PROGRAMMED AGE
gateway.gateway.networking.k8s.io/main-gateway traefik True 10s

NAME HOSTNAMES AGE
httproute.gateway.networking.k8s.io/whoami-route 10s

Troubleshooting

Common errors and fixes
GatewayClass ACCEPTED: Unknown
Cause: CRD version mismatch or Gateway provider not enabled.
Run kubectl logs -n kube-system deploy/traefik | grep -i gateway to confirm. Install v1.5.1 CRDs and ensure --providers.kubernetesgateway is in Traefik's args.
Cannot find entryPoint for Gateway: no matching entryPoint for port 80
Cause: Gateway listener port set to 80 but Traefik's web entrypoint runs on 8000 internally.
Fix: Change port: 80 to port: 8000 in your Gateway spec.
Failed to watch *v1.BackendTLSPolicy: the server could not find the requested resource
Cause: Only standard CRDs installed — BackendTLSPolicy is in the experimental channel.
Fix: Apply experimental-install.yaml from Gateway API v1.5.1 release.
curl: (7) Failed to connect to localhost port 8080
Cause: k3d cluster not running or port mapping not set up.
Check k3d cluster list and docker ps | grep k3d. Recreate cluster with --port "8080:80@loadbalancer".
404 page not found
Cause: Route not attached to Gateway or Gateway not programmed.
Check kubectl describe gateway main-gateway -n demo for conditions. Ensure parentRefs in HTTPRoute points to the correct Gateway name and namespace.

bash debug toolkit 
# Check Traefik is receiving Gateway events
